## 🏗️ 시스템 5대 도메인

| # | 도메인 | 설명 |
|---|--------|------|
| 1 | 기관(고객) 관리 | 한의원 정보, 납품지, 결제조건, 담당자 |
| 2 | 상품(품목) 관리 | 소모품·의료기기 카탈로그, 단가 |
| 3 | 주문 관리 | 주문 접수 → 확인 → 출고 → 납품 |
| 4 | 납품 관리 | 납품서, 수령확인, 거래명세서 발행 |
| 5 | 정산·청구 관리 | 세금계산서, 미수금, 입금확인, 리포트 |

---

## ✅ MVP 우선순위 (1단계)

- [ ] 한의원 등록/관리
- [ ] 품목 등록/관리
- [ ] 주문 접수 및 상태 관리
- [ ] 납품서 생성 (PDF)
- [ ] 월별 청구 내역 조회

## 🔧 기술 스택 (예정)

- **Frontend**: React + TypeScript + Tailwind CSS
- **Backend**: Node.js (Express) or Python (FastAPI)
- **DB**: PostgreSQL
- **배포**: 추후 결정

---

## ❓ 결정 필요 항목

1. 운영 주체: 공급사 단독 vs 멀티 공급사 플랫폼
2. 세금계산서 발행: 시스템 내 직접(홈택스 연동) vs 외부 수동
3. 모바일 앱 필요 여부 (한의원 원장 직접 주문)
4. 의료기기 등급 포함 여부 (1~4등급 이력관리 의무)
