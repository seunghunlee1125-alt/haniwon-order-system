# 05. 기술 스택 및 개발 계획

## 권장 기술 스택

### Frontend
| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | React + TypeScript | 컴포넌트 재사용, 타입 안정성 |
| UI 라이브러리 | Tailwind CSS + shadcn/ui | 빠른 UI 개발 |
| 상태관리 | Zustand or React Query | 서버 상태 캐싱 |
| PDF 생성 | react-pdf or jsPDF | 거래명세서 |
| 차트 | Recharts | 매출 리포트 |
| 라우팅 | React Router v6 | |

### Backend
| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Node.js + Express or FastAPI(Python) | REST API |
| ORM | Prisma (Node) or SQLAlchemy (Python) | DB 추상화 |
| 인증 | JWT + Refresh Token | |
| 파일저장 | AWS S3 or 로컬 스토리지 | 서명 이미지, PDF |
| 이메일 | Nodemailer or SendGrid | 거래명세서 전송 |

### Database
| 항목 | 선택 |
|------|------|
| RDBMS | PostgreSQL |
| 개발 환경 | Docker + docker-compose |

### 배포 (추후 결정)
- 소규모: Vercel (frontend) + Railway/Render (backend)
- 중규모: AWS EC2 + RDS + S3

---

## 개발 단계

### 1단계 — MVP (약 6~8주)
- [ ] 프로젝트 초기 설정 (폴더구조, lint, prettier)
- [ ] DB 스키마 생성 및 마이그레이션
- [ ] 인증 (로그인/로그아웃, JWT)
- [ ] 한의원 CRUD
- [ ] 품목 CRUD
- [ ] 주문 접수 및 상태 관리
- [ ] 납품서 생성 + PDF 출력
- [ ] 월별 청구서 생성
- [ ] 미수금 조회

### 2단계 — 고도화 (약 4~6주)
- [ ] 한의원 로그인 포털 (직접 주문)
- [ ] 카카오톡/SMS 알림 연동
- [ ] 전자서명 수령 확인
- [ ] 고객별 계약단가
- [ ] 재고 관리
- [ ] 매출 리포트 (차트)

### 3단계 — 추가기능
- [ ] 세금계산서 홈택스 연동
- [ ] 의료기기 이력 관리
- [ ] 영업사원 모바일 앱
- [ ] 연체 자동 알림

│   │   │   ├── deliveries.js
